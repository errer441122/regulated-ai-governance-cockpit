# Project Positioning - Ducati Data Analyst Customer Experience

## Target role

Ducati Motor Holding - Tirocinio in Data Analyst (Customer Experience), Bologna.

## What the job asks for

The posting emphasizes customer behavior analysis, customer satisfaction reporting, customer segmentation/profiling, trend identification, cross-functional collaboration, Excel/Power BI or data-analysis tooling, clear communication of complex data and customer focus.

Source: https://it.indeed.com/viewjob?jk=40f0dba77d577c92

## How to present the project

Short version:

> I created a Customer Experience analytics cockpit as a portfolio case study. It uses simulated data to show how I would collect and organize customer signals, build dashboard views, segment customers, identify satisfaction trends and translate insights into action-ready recommendations for CX, CRM, dealer, service and product stakeholders.

## Best sections to show first

1. `Case Study Brief` - explains the CX problem, data workflow and limits.
2. `CX Analytics Cockpit` - shows feedback volume, weighted insight volume, segment scoring and journey workflow.
3. `Customer Segment Scoring` - demonstrates segmentation, profiling and prioritization logic.
4. `Customer Insight Brief Simulator` - shows structured communication of findings with human review.
5. `Insight-to-Action Tracker` - shows privacy-aware data handling, stakeholder mapping and action tracking.
6. `Reporting Guide` - shows how the reporting workflow could be explained to CX stakeholders.
7. `Dashboard MVP Brief` - frames the cockpit as a realistic reporting product.
8. `Methodology` - makes clear that all data is simulated and not confidential.

## Suggested explanation in interview

I would frame it as an analytics and communication project, not as a production system. The value is in the logic: source mapping, data-quality thinking, segmentation, dashboarding, insight communication and follow-up ownership. If I had real approved CX data, the next step would be to validate sample sizes and missing values, build a Power BI-style report, compare satisfaction by journey stage and measure whether actions improve the customer signal over time.

## Added tooling evidence

The project includes an Excel dashboard prototype using the same simulated CX dataset. It directly supports the Excel/Power BI requirement in the announcement while keeping the web cockpit as the portfolio presentation layer.
