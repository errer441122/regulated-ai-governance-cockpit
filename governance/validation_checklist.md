# Validation Checklist

## Pre-Release Checks

| Check | Evidence | Status |
| --- | --- | --- |
| Data source documented | `credit-risk-model-risk-lab/README.md` | Present |
| Target definition documented | `credit-risk-model-risk-lab/reports/model_card.md` | Present after evaluation run |

| Human-review boundary stated | `README.md`, model cards, API metadata | Present |
| Production limitation stated | `README.md`, `EVIDENCE_MAP.md`, model cards | Present |

## Release Gate

The artifact can be described publicly only if:

- the exact public data source is named
- model-risk limitations are visible
- metrics are generated by executable scripts
- no employer/client/system ownership is implied
- no production credit or compliance claim is made

## Manual Gate For Live API

The API can be described as live only after:

- a public base URL is available
- `/health` responds with `{"status": "ok"}`
- `/metadata` returns the scope and limitations
- `/docs` displays OpenAPI docs
- `evidence/deployment_evidence.md` records URL, timestamp, commit SHA, and curl evidence
